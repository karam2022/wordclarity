# Gotcha catalog

Twelve trap doors observed across four reference migrations
(`paritytech/sh33ts#33`, `paritytech/t3ams#30`, `paritytech/t3rminal#98`,
w3s-conference-app Phase 1). Apply the fix when the symptom appears;
reference the gotcha number (G1–G12) from the migration spec when applicable.

| # | Gotcha | Symptom | Cause | Fix |
|---|---|---|---|---|
| G1 | JSON-RPC provider hoist | `TypeError: onReady is not a function` at first WebSocket connect | `node-linker=hoisted` / bun isolated install picks up `@polkadot-api/json-rpc-provider@0.0.4` (empty `"main"` stub) or `json-rpc-provider-proxy@0.2.8` (legacy no-arg `input()`) as transitive | Add `pnpm.overrides` (or `overrides` for bun/npm). `@polkadot-api/json-rpc-provider: ^0.2.0` is always required — it is also present at the SDK monorepo root (`product-sdk/package.json`). `@polkadot-api/json-rpc-provider-proxy: ^0.4.0` is also needed when the legacy `0.2.8` proxy is being hoisted (add it; it is **not** in the SDK monorepo root). |
| G2 | "No signer available" on Cloud Storage upload | Cloud Storage upload fails after account is connected | `CloudStorageClient.create` uses `createLazySigner` against whichever signer source it's given. When using the App singleton (`app.cloudStorage`), the underlying lazy signer pulls from `app.wallet`'s account — it's populated only after `app.wallet.connect()` + `app.wallet.selectAccount(addr)` | Two valid fixes: (a) call `app.wallet.connect()` + `selectAccount(addr)` after the existing account-connection flow when using the App-bound cloud storage client; (b) construct `CloudStorageClient.create({ environment, signer })` standalone with an explicit signer / `createLazySigner` — bypasses the App-singleton plumbing entirely |
| G3 | HKDF info-string mismatch | Existing on-chain entries fail to decrypt after migrating to `KeyManager.deriveKeypairs()` | SDK hardcodes `info: 'encryption-keypair'` / `'signing-keypair'`; legacy implementations often used app-specific infos (e.g. `'dotdocs-encryption-keypair'`) → output bytes differ | Byte-for-byte comparison before adopting. If mismatch and there's persisted ciphertext: **defer this sub-area** and open a follow-up for a key-rotation migration. `KeyManager.fromSignature` + `deriveSymmetricKey('domain:'+id)` is safe — info is caller-controlled. |
| G4 | PAPI 2.x event watch shape change | `api.event.X.watch(filterFn)` no longer exists; runtime breakage | In PAPI 2.x, `watch()` emits `{ block, events[] }` per finalized block; filter is applied in the subscriber, not as an argument | Rewrite the subscriber to iterate `events[]` and filter inside |
| G5 | `Binary` class → namespace | `bytes.asHex()` / `Binary.fromBytes(uint8)` no longer exist | In PAPI 2.x, `Binary` is a namespace, not a wrapper class | `Binary.toHex(uint8)` for encoding; raw `Uint8Array` for payloads (no more wrapping) |
| G6 | `ws-provider/web` import path | Build / typecheck breaks on `polkadot-api/ws-provider/web` | Path removed in PAPI 2.x | Replace with `polkadot-api/ws` (legacy-spec compat is now bundled); drop explicit `withPolkadotSdkCompat` wrapper |
| G7 | `cloudStorage: false` required when not using Cloud Storage | Unnecessary WebSocket connection at boot | `createApp` default is `cloudStorage: { environment: 'paseo' }` | Pass `cloudStorage: false` explicitly |
| G8 | Container-only vs dual paths | SDK doesn't work standalone in some cases | Some App hooks (e.g. `app.cloudStorage`) are container-only by design | Gate on `isInsideContainer()` / `isInsideContainerSync`; keep standalone PAPI raw paths for CLI scripts and web-only deployments |
| G9 | `StoreResult` no longer exposes `blockHash` | Code reading `result.blockHash` breaks | SDK 0.2+ removed the field | Reconstruct via `api.query.System.BlockHash.getValue(result.blockNumber)` |
| G10 | `.papi/descriptors/package.json` bump forgotten | Type mismatches against generated descriptors | The descriptors package version must follow `polkadot-api` major bumps | Always touch `.papi/descriptors/package.json` alongside the `polkadot-api` bump |
| G11 | `@novasamatech/*` removed but reappears | Lockfile still shows a `@novasamatech/*` package after migration | `@parity/product-sdk-host` re-exports `getTruApi` / `getPreimageManager` from `@novasamatech/host-api-wrapper` — it remains as a transitive dep. (Legacy products imported this as `@novasamatech/product-sdk` before its rename in PR #110.) | Expected. Remove only as a **direct** dependency (whichever name the legacy product used). Document in PR body that it remains transitive via host under its current name. |
| G12 | `cidToBytes` / `cidFromBytes` are full-CID helpers, not digest helpers | Runtime error on first round-trip: `Expected 32-byte digest, got 38`; or hash-storage extrinsic submits 38-byte hashes instead of 32 | The upstream `@parity/bulletin-sdk`'s `cidToBytes(cid): Uint8Array` returns the full CIDv1 binary encoding (1+1+1+1+32 = 38 bytes for blake2b-256 raw codec), not the 32-byte multihash digest. Symmetric for `cidFromBytes`. The names invite misuse for products storing digests in `bytes32` on-chain slots. | Use `cidToPreimageKey(cid.toString()): \`0x${string}\`` and `parseCid(hashToCid(bytes32))` from `@parity/product-sdk-cloud-storage` directly. Both are digest-level. |
